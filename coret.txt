## 3. Why Playwright Cannot Run (Root Cause & Solution)
*   **Root Cause:**
    Playwright automatically downloads its own precompiled driver and browser binaries (like Chromium, Firefox, etc.) to the user cache directory: `/home/lintang/.cache/ms-playwright/`.
    Because NixOS does not conform to the Filesystem Hierarchy Standard (FHS) and lacks a standard dynamic linker path (like `/lib64/ld-linux-x86-64.so.2`), standard dynamically linked binaries cannot be executed out-of-the-box, resulting in a dynamic linker loading failure:
    ```text
    Could not start dynamically linked executable: /home/lintang/.cache/ms-playwright/chromium-1228/chrome-linux64/chrome
    NixOS cannot run dynamically linked executables intended for generic linux environments out of the box.
    ```
    This causes the subagent driver to fail with exit status 127.

*   **Solutions/Workarounds:**
    1.  **Use System-provided Chromium:**
        Set the following environment variables when running Playwright:
        ```bash
        export PLAYWRIGHT_SKIP_BROWSER_DOWNLOAD=1
        export PLAYWRIGHT_CHROMIUM_EXECUTABLE_PATH=$(which chromium)
        ```
        The Nix environment already has Chromium installed and working at:
        `/nix/store/pzngcmr4kgrw06vlj9cp7pjif3mrcwyh-chromium-150.0.7871.124/bin/chromium`
    2.  **Enable nix-ld:**
        Configure the NixOS system to use `nix-ld` to stub dynamic link libraries so that generic Linux binaries can execute normally.
